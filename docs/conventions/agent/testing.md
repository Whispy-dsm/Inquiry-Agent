# Agent Testing Convention

## 필수 테스트 규칙

- 새 문의 처리 흐름을 추가하면 정상, 경계, 실패, 에스컬레이션 평가 케이스를 작성합니다.
- 분류 기준을 수정하면 기존 문의 유형의 회귀 평가를 실행합니다.
- 프롬프트를 수정하면 대표 문의와 실패했던 문의를 다시 검증합니다.
- 도구 연동을 수정하면 성공, 실패, timeout, 권한 오류 케이스를 검증합니다.
- 개인정보나 정책 관련 로직은 우회 사례를 포함해 테스트합니다.

## 평가 케이스 구조

평가 케이스에는 아래 항목을 포함합니다.

- 사용자 문의
- 채널 또는 맥락
- 기대 분류
- 필요한 근거 또는 도구
- 기대 행동: 자동 응답, 확인 질문, 에스컬레이션
- 금지 행동
- 허용 가능한 응답 기준

## 검증 기준

- 변경 범위에 가장 가까운 평가를 먼저 실행합니다.
- 공통 프롬프트, 분류기, 도구 호출, 지식 검색을 건드렸다면 전체 회귀 평가를 검토합니다.
- 모델 출력은 정확성, 근거성, 정책 준수, 톤, 다음 행동 명확성으로 확인합니다.
- 테스트를 실행하지 못한 경우에는 완료 보고에 이유와 남은 위험을 명시합니다.
