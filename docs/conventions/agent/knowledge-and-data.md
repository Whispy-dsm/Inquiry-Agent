# Agent Knowledge and Data Convention

## 지식 소스 원칙

- 에이전트가 답변에 사용하는 지식 소스는 위치, 소유자, 갱신 주기, 신뢰도를 문서화합니다.
- 오래된 문서와 최신 문서가 충돌하면 최신 기준을 확인하고 충돌 사실을 기록합니다.
- 제품 정책, 가격, 법적 문구, 보안 안내처럼 자주 바뀌는 정보는 최신성 확인 절차를 둡니다.
- 근거가 없는 답변을 만들지 않습니다.

## 검색과 근거

- 문의 유형별로 우선 검색할 지식 소스를 정합니다.
- 검색 결과는 답변에 필요한 부분만 요약해 사용합니다.
- 여러 근거가 충돌하면 자동으로 단정하지 않고 확인 또는 에스컬레이션합니다.
- 사용자에게 근거를 보여줄 수 있는 채널이면 출처나 확인 기준을 함께 제공합니다.

## 개인정보와 민감 정보

- 문의 원문에는 개인정보나 민감 정보가 포함될 수 있다고 가정합니다.
- 처리에 필요 없는 정보는 저장하지 않습니다.
- 로그와 평가 데이터에서는 개인정보를 마스킹하거나 합성 데이터로 대체합니다.
- 외부 도구에 전달하는 입력은 최소화합니다.
- 삭제 요청, 보안 사고, 법적 요청은 자동 처리하지 않고 정책에 따라 에스컬레이션합니다.

## 상태와 메모리

- 단기 상태는 현재 문의 해결에 필요한 범위로 제한합니다.
- 장기 기억은 목적, 사용자 동의, 보존 기간, 삭제 방법이 명확할 때만 사용합니다.
- 모델이 추론한 사용자 속성을 사실처럼 저장하지 않습니다.
- 상태 저장 구조를 바꿀 때는 기존 대화와 평가 케이스에 미치는 영향을 확인합니다.

## 평가 데이터

- 실제 문의를 평가 데이터로 사용할 때는 개인정보를 제거합니다.
- 평가 케이스에는 입력 문의, 기대 분류, 기대 행동, 허용 응답 범위를 포함합니다.
- 실패 사례는 재발 방지용 회귀 케이스로 추가합니다.
